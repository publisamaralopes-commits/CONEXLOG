IMPLEMENT PROFESSIONAL FIRST-LOGIN PASSWORD FLOW

Create a secure first-login password setup system for employees.

==================================================
USER CREATION FLOW
==================

When an administrator creates a new employee:

The admin defines:

* Nome
* Usuário
* Senha Provisória
* Cargo/Função
* Tipo de usuário

Example:
Temporary password:
123456

==================================================
FIRST LOGIN BEHAVIOR
====================

When the employee logs in for the FIRST TIME:

DO NOT allow direct access to the dashboard yet.

Instead:

* redirect to "Create New Password" screen
* require the employee to define a personal password

==================================================
REQUIRED SCREEN
===============

Fields:

* Senha Provisória
* Nova Senha
* Confirmar Nova Senha

Rules:

* minimum password length
* password confirmation validation
* secure password hashing
* clear success/error messages

==================================================
AFTER PASSWORD CHANGE
=====================

After successful password creation:

* invalidate temporary password
* mark user as "passwordConfigured = true"
* redirect to dashboard
* create normal authenticated session

==================================================
SECURITY REQUIREMENTS
=====================

* passwords must always be encrypted
* temporary password must never appear in UI after creation
* prevent reuse of temporary password
* protect routes
* validate sessions
* prevent bypassing first-login flow

==================================================
ADMIN FEATURES
==============

Administrator must be able to:

* reset employee passwords
* generate new temporary passwords
* force password reset again if needed

==================================================
DATABASE
========

Add fields like:

* temporaryPassword
* passwordConfigured
* mustChangePassword
* lastPasswordChange

==================================================
FINAL RESULT
============

The CONEX authentication system must behave like a professional enterprise platform with secure employee onboarding.
